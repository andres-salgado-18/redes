#### Ejercicio 1

Un puerto es un número (generalmente de 16 bits) que permite identificar  de forma única a un proceso de un host. Este número, se usa junto a la dirección IP para determinar qué proceso recibe la información. Lo cual, añade multiplexación y demultiplexación al protocolo IP, brindado la capacidad de que varios procesos compartan simultáneamente un conexión entre sí.

En el caso de no implementar este concepto, no podría haber comunicación end-to-end proceso a proceso, debido a que sería imposible determinar qué proceso corresponde los datos enviados, además de limitar las conexiones de un host a una.

UDP es básicamente IP, agregando esta demultiplexación, por lo que se podría ver como una implementación minimalista de comunicación proceso a proceso. Además de conservar las características de conexión de IP como
- best effort
- NO orientado a conexión
Con las ventajas 
- poco overhead en el header y en establecimiento de comunicación
- puede manipular gran cantidad de clientes en simultáneo
- control de información enviada y recibida
y desventajas
- congestión de envío/recibo
- no garantiza orden ni confiabilidad
- pérdida de información


Por otro lado TCP provee un servicio con muchas más garantías que UDP, como
- orientado a conexión
- confiable
- byte-stream
- con control de flujo, usando una versión de sliding window
- ordenado
Lo anterior a cambio de mucho mayor complejidad de implementación y overhead en intercambio de paquetes.

Casos de uso
- Para aplicaciones donde no es necesario entregar todos los datos de manera correcta y se requiera baja latencia es ideal el uso de UDP (ej. streaming).
- Para aplicaciones donde sea esencial la confiabilidad e integridad de los datos es ideal el uso de TCP (ej. HTTP)
